# Task List — Advanced Enterprise Investigation UI

Source of truth: `requirements.txt` (Phase 21 — Advanced Enterprise UI) + `CLAUDE_INSTRUCTIONS.md` (operating model & architecture rules).

## Context & Key Decision

An existing full-stack "Data Lineage Platform" (`lineage-backend` FastAPI + MSSQL + Neo4j, `lineage-frontend` React/Vite) is present. `requirements.txt` — the literal source-of-truth pointer — specifies the **Advanced Enterprise Investigation UI**. Per `CLAUDE_INSTRUCTIONS.md` §17 ("do not rewrite the whole project") we **extend the existing frontend** rather than rebuild.

To satisfy the "self-contained / mock-testable / no mandatory external dependency except the LLM" rules, the new UI is backed by a **typed, config-driven mock data layer** (`VITE_DATA_MODE=mock|api`) so it is fully demonstrable and testable without standing up MSSQL/Neo4j. The mock domain is deliberately generic (no hardcoded regulatory concepts) per §5 (no project-specific hardcoding).

Signature design element: a **provenance / trust-label system** (Parsed from code · Derived from graph · Generated by AI · Inferred · Added by user · Verified by reviewer) threaded through every panel.
